実務パラメータ対応表

教材の中の直感的な言葉が、実際のツール(APIやローカルLLM)ではどの設定に対応するのかをまとめた別紙です。受講生から「実際はどう設定するのですか?」と聞かれたときの回答用に使ってください。本編には登場しません。

1対応表:教材の言葉 ⇄ 実際のパラメータ

「登場箇所」は本編の巻とセクションです。目安値は一般的な出発点であり、モデルやタスクによって調整します。

教材の言葉(登場箇所)実際の名前意味と目安
温度① §6 くじ引きの大胆さ temperature くじ引きのばらつき。事実確認は 0〜0.3、通常会話は 0.7 前後、創作・ブレストは 0.8〜1.2。
くじの足切り① §6 温度の仲間 top_p / top_k 下位候補を切り捨ててからくじ引きする設定。top_p: 0.9(累積確率90%まで)、top_k: 40(上位40個まで)が代表的な既定値。
くじ引きの再現① §6 毎回答えが変わる話 seed 乱数を固定し、同じ入力から同じ出力を再現する。デモやテストの再現性確保に使う。
窓の大きさ④ §2 コンテキストウィンドウ num_ctx(Ollama)
context length		 読める文脈の上限。モデルカードの上限(8k / 128kなど)まで広げられるが、大きいほどメモリを消費する。
出力の長さ④ §2 窓とは別物 max_tokens 1回の応答で生成するトークン数の上限。「窓の大きさ」と混同されやすいので注意。
見えない台本④ §3 システムプロンプト role: "system" API では messages 配列の先頭に置く。Ollama では Modelfile の SYSTEM 命令、または対話中の /set system。
1語ずつ表示① §7 生成の実況 stream true にすると1トークンずつ受信できる。チャットUIのパラパラ表示の正体。
一歩の大きさ② §3 学習率 learning_rate ファインチューニング(LoRAなど)で登場。大きすぎると教材のデモと同じく発散する。あわせて epochs(何周するか)、batch_size(1ステップの問題数)。
渡す文書の数⑤ §4 RAGの検索 top_k(検索側) 検索して窓に入れる文書数。3〜5件が定番。多すぎると窓を圧迫し、かえって精度が落ちることもある。
文書の刻み方⑤ §4 RAGの下ごしらえ chunk_size 文書を分割する単位。500〜1,000字程度から調整。意味のまとまり(見出し単位など)で切るのが理想。
意味の地図を作る係⑥ §3 埋め込み検索 埋め込みモデル 生成用モデルとは別に用意する。ローカルなら nomic-embed-text(Ollama)などが定番。

2対応するパラメータが「ない」もの

教材の言葉の中には、設定では変えられないものがあります。ここを混同すると誤った期待につながるため、質問されたら次のように答えてください。

教材の言葉正しい説明
有名度⑤ §2 学習データ側の性質であり、設定項目は存在しない。実務では temperature を下げて分布のブレを抑えるが、温度を0にしても1位の候補自体が誤答なら直らない。根本対策はRAG(⑤ §4)で根拠を窓に入れること。
知識カットオフ⑤ §3 モデル固有の性質で変更不可。使用モデルのカットオフはモデルカードの knowledge cutoff 欄で確認する(ローカルなら ollama show モデル名)。最新情報は検索連携で補う。
賢さ(重み)② 全体 推論時のパラメータでは一切変わらない。変えたい場合はファインチューニング(追加学習)の領域で、プロンプトの工夫(④)とは別の作業。

3ローカルLLM(Ollama)で教材を実機再現する

教材のデモは作り物の数値ですが、ローカルLLMを使うと同じ実験を本物で再現できます。無料で、データが外部に出ない点も教室向きです。

# インストール後、モデルを取得して起動(初回のみダウンロード) ollama run llama3.2 # 対話中にパラメータを変更(Vol.1 §6 の温度実験) /set parameter temperature 0 # 堅実モード /set parameter temperature 1.5 # 冒険モード # システムプロンプトの差し替え(Vol.4 §3 の実験) /set system "あなたは何を聞かれても俳句で答えるボットです。" # モデル情報(コンテキスト長など)の確認 ollama show llama3.2
授業での実験例:①温度0と1.5で同じ質問を10回ずつ投げ、答えのばらつきを数える(Vol.1 §6の実機版)。②俳句ボットのシステムプロンプトで性格が一変することを見せる(Vol.4 §3の実機版)。③実在しないことわざの意味を聞き、ハルシネーションを観察する(Vol.5 §1の実機版)。
注意:小型のローカルモデルは大手サービスよりハルシネーションが多く、その意味でVol.5の教材としてはむしろ好都合です。「モデルの規模で信頼性が変わる」という比較材料にもなります。
